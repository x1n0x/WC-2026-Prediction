# WC 2026 Match Outcome Predictor

Predicts **Win / Draw / Loss** outcomes for the remaining FIFA World Cup 2026
matches (Matchdays 2 & 3 + knockout stage) from historical World Cup data and
**Matchday-1 results used as live form features**.

---

## 1. Problem statement

Given the 48 qualified teams, the group draw, and the 24 Matchday-1 results, produce
calibrated probabilistic forecasts for every remaining match:

* **Train** on World Cup history (1930–2022).
* **Validate** on Matchday 1 (24 matches) — these are *held out* as ground truth.
* **Predict** Matchday 2 (primary deliverable), Matchday 3, and a Monte-Carlo
  projection of group advancement and the eventual champion.

Outcome is always coded **from the home team's perspective**:
`0 = home_win`, `1 = draw`, `2 = away_win` → `(p_home_win, p_draw, p_away_win)`.

## 2. Data sources

| # | Dataset | Used for | Kaggle |
|---|---------|----------|--------|
| 1 | `matches_1930_2022.csv` | historical WC results → win-rate, H2H, Poisson/BT strengths | `piterfm/fifa-football-world-cup` |
| 2 | `elo_ratings_wc2026.csv` | team Elo (strength prior) | `afonsofernandescruz/2026-fifa-world-cup-historical-elo-ratings` |
| 3 | `wc_2026_fixtures.csv`, `wc_2026_teams.csv` | schedule, FIFA rank, confederation | `kulkarniparth09/fifa-world-cup-complete-dataset-19302026` |
| 4 | `team_dataset.csv` | squad age, market value, recent form | `harrachimustapha/fifa-world-cup-team-dataset` |

> **Data note.** The pipeline is **real-data-first with a deterministic synthetic
> fallback** (`SEED=42`). With the four Kaggle CSVs in `data/raw/` (the configured
> setup), `src/ingest.py` uses them: real Elo (125-year time series → *year-appropriate*
> Elo per historical match), real FIFA ranks, real squad market value / age /
> appearances, and **964 real WC matches (1930–2022)**. If a file is missing it
> synthesizes an equivalent table so the pipeline still runs with no credentials.
> Every choice is logged (`[real]` vs `[synthetic]`).
>
> Two deliberate exceptions keep the project internally consistent with the task:
> * **Group composition** stays as defined in `src/config.py`. The Kaggle
>   `wc_2026_teams.csv` / `wc_2026_fixtures.csv` use a *different* draw (groups
>   G↔H and K↔L are relabelled and recomposed), which contradicts the task's
>   hardcoded MD1. Confederations and FIFA ranks (group-independent) are still
>   merged from the real file.
> * **MD2/MD3 fixtures** are generated by the standard round-robin rotation over
>   the task's groups (a consistency guard rejects the real fixture file when its
>   groups disagree, falling back automatically and logging why).
>
> Matchday-1 results (the validation set) are hardcoded in `src/config.py`.

## 3. Feature list

Built per match in `src/features.py`, **leakage-safe** (see §6).

**Strength signals**
* `elo_diff` = home_elo − away_elo
* `fifa_rank_diff` = away_rank − home_rank (positive ⇒ home better ranked)
* `market_value_ratio` = log(home_value / away_value)
* `squad_age_diff` = home_avg_age − away_avg_age

**Live form (Matchday 1)** — for training, the *previous match in the same
tournament* (MD1 is just the special case for MD2/MD3, so train & predict share
one schema):
* `home_md1_goals_scored`, `home_md1_goals_conceded`, `home_md1_points`
* `away_md1_goals_scored`, `away_md1_goals_conceded`, `away_md1_points`
* `form_diff` = home_md1_points − away_md1_points

**Historical (no leakage — only matches before the current one)**
* `home_historical_win_rate`, `away_historical_win_rate`
* `h2h_home_win_rate` (NaN if teams never met → median impute)
* `home_wc_appearances`, `away_wc_appearances`

**Context**
* `is_home_host` (1 if USA / Canada / Mexico)
* `conf_home_*`, `conf_away_*` — one-hot confederation (UEFA/CONMEBOL/CAF/AFC/CONCACAF/OFC)
* `same_confederation`
* `stage_ord` — ordinal (group=0 … final=5)

## 4. Models

| Model | Idea |
|-------|------|
| **Baseline** | always predict the most frequent training class |
| **Poisson** | team attack/defense strengths via a Poisson GLM; λ = exp(intercept + attack − defense + home_adv); W/D/L from **10,000 simulated score lines** per fixture |
| **LightGBM** | 3-class classifier, `objective=multiclass`, `class_weight=balanced`, **Optuna-tuned** (50 trials, stratified 5-fold CV on multi-logloss) |
| **Bradley-Terry** | latent team strengths via logistic regression on decisive historical results; strength diff → logistic win prob (draws via empirical base rate) |

**Ensemble** = weighted average of Poisson + LightGBM probabilities; the weight is
tuned on the MD1 validation set by minimizing log-loss.

## 5. Results (MD1 validation, 24 matches)

Real-data run (`SEED=42`), trained on the 410 historical WC matches contested
between teams that are in the 2026 field:

| model | accuracy | log-loss | Brier |
|-------|---------:|---------:|------:|
| baseline | 0.500 | 1.070 | 0.640 |
| **poisson** | **0.542** | **0.973** | **0.598** |
| bradley_terry | 0.458 | 1.071 | 0.651 |
| lightgbm | 0.417 | 1.314 | 0.783 |
| **ensemble** | **0.542** | **0.973** | **0.598** |

* The ensemble weight-tuning landed on **100% Poisson / 0% LightGBM**: with the
  mandated `class_weight='balanced'` (which trades probability calibration for
  balanced accuracy) and only ~410 noisy training matches, the LightGBM
  probabilities are poorly calibrated (log-loss 1.31), so the optimizer correctly
  down-weights it to zero. The ensemble therefore equals the Poisson model here —
  a legitimate finding, not a bug. All four models are still trained, saved, and
  compared.
* **ROI back-test** (flat stake on the ensemble's top-50% confidence picks vs
  hardcoded pre-tournament odds): **−1.9%** over 12 bets (MD1 had several upsets,
  e.g. Iran 2-2 NZ, Saudi 1-1 Uruguay, Belgium 1-1 Egypt).
* **Monte-Carlo title favourites** (2,000 sims): Spain ≈ 19%, Argentina ≈ 14%,
  France ≈ 11%, then Netherlands / England / Norway / Germany / Brazil / Colombia.

> Small-sample caveat: 24 validation matches make these numbers noisy. The
> framework and the calibrated probabilities, not the exact percentages, are the
> deliverable.

Plots: `plots/confusion_matrix.png`, `plots/calibration_curve.png`,
`plots/feature_importance.png`.

## 6. Code-quality guarantees

* **No leakage** — features for match *M* use only data available before *M*:
  win-rate/H2H use a strict year cutoff; pre-match form uses only earlier matches
  of the same tournament; MD1 validation form is zero (tournament openers);
  `elo_diff` for a historical match uses the **Elo snapshot at that match's year**
  (the 125-year series), not today's Elo. FIFA rank / market value / squad age are
  available only as current (2026) team values and are used as static team priors
  on historical rows — a documented simplification (a feature, never the target).
* **Missing values, per column**
  * `h2h_home_win_rate`: NaN when teams never met → **median impute** (in the LGBM
    pipeline). Tree models receive NaNs natively elsewhere.
  * `home/away_historical_win_rate`: NaN for teams with no prior history →
    median impute.
  * unknown teams in the Poisson/BT models fall back to a 0 strength contribution.
* **Fixed seeds** — `SEED = 42` everywhere (numpy RNGs, LightGBM, Optuna sampler,
  CV splits, simulations).
* Every `src/` module has a `if __name__ == "__main__"` smoke test.
* Pinned `requirements.txt`.

## 8. How to run

```bash
python -m venv .venv
.venv\Scripts\activate            # Windows  (source .venv/bin/activate on *nix)
pip install -r requirements.txt

# (optional) drop the four Kaggle CSVs into data/raw/ for real data

python -m src.config        # sanity-check the hardcoded reality
python -m src.ingest        # build data/processed/*.csv
python -m src.features      # build & summarize the training table
python -m src.model         # train all 4 models, write metrics + plots + models/
python -m src.predict       # write predictions/ + dashboard.html
python -m src.dashboard     # (re)build the dashboard from existing predictions
```

`python -m src.predict` runs the full chain (train → predict → dashboard) on its
own. Then open **`dashboard.html`** in any browser — a self-contained,
broadcast-style view of every forecast. A top-right **EN / RU** switch toggles
the whole page (UI copy, team names, model names) and remembers the choice;
it opens in Russian automatically on Russian-language browsers. No server, no
build step.

### Primary output — `predictions/matchday2_predictions.csv`

```
match_id | home_team | away_team | p_home_win | p_draw | p_away_win | predicted_outcome | confidence | model
```
